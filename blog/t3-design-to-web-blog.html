What a responsive site is, and why responsiveness is important.

A responsive site is one that will display differently depending on what device you are looking at the website from. This is important because a website needs to look good and display the necessary information whether you are using your phone, desktop or any other device.

What mobile first design is, and why it's important.

Mobile first design is designing website primarily to work on mobile phone, then adapting it to display differently on larger devices. This is a good way to design because it is a lot easier to add more stuff and features to a larger screen than it is to try and squeeze everything from large screen into a mobile screen.

What frameworks are, and their pros and cons.

Frameworks are pre written groups of general code. They can be added to your work so you can use code inside them.

Pros of frameworks are its easier and faster to use code that has already been written. This saves reinventing the wheel and you get to borrow lots of clever tricks from experienced developers.

Cons of frameworks can be that you are loading lots of unused code from the framework, slowing loading times. Sometimes things in the framework may not be exactly as you want and you need to write code to override it. You don’t learn as much about how things work using frameworks than if you built it yourself.

What a wireframe is and why we use it.

Wireframe is the use of a grid of columns and rows used to place elements where we want on a webpage. We use this because it is easier and more versatile than placing elements in relation to each other. A wireframe structure is very good at responding to different screen sizes.

The aspects of your wireframes you found difficult to implement, and why.

The wire frames didn’t seem to be working as i thought they would when I first started shaping them. I added borders around everything and this made it much easier to see what was going on. After a lot of playing around I had everything in the right place and formatting as I wanted. Once the borders were removed it all went wrong. Worked out that blank rows I had been adding between things for spacing were only displaying because of the borders. I had a play around to get them to display but the code got quite messy so I removed the blank rows and used margins instead.

After everything looked right and tested fine using display emulators I made the site live and checked it on my phone. And the webpage displayed in desktop view. After more playing around and more reading, I found that some phones will display a webpage using their full resolution rather than the scaled CSS resolution unless there is a ‘Viewport’ meta tag in the head of the HTML. I added this to all my HTML pages and all looked as it should when viewing on my phone.
